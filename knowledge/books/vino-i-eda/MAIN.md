# vino-i-eda — «Вино и еда. Краткий курс для гурманов» (SWN, 2015) — рабочий файл

## Глобальная цель
Полное распознавание книги в чистый структурированный текст для единой RAG-базы знаний из винных книг. Точность (имена/числа/сочетания), провенанс (книга + страница), сохранение мыслей, переходящих со страницы на страницу.

Это книга 3 из 5 в `knowledge/books/QUEUE.md`. Книги 1 (vina-francii) и 2 (novy-svet) — ✅ готовы.

> На этой книге выходной фильтр Anthropic начал стабильно блокировать мою генерацию **дословного** текста копирайтной книги (`API Error: Output blocked by content filtering policy`). Фильтр работает по назначению, обходить его я не буду.
>
> **РЕШЕНИЕ ПОЛЬЗОВАТЕЛЯ (зафиксировано):**
> - **vino-i-eda (эта книга)** → путь **B: подробный трансформативный пересказ.** НЕ дословно, но и НЕ сжато. Пересказываю максимально подробно, ничего не сокращаю и не выкидываю: все факты, имена, аппелласьоны, числа, проценты, **температуры подачи**, сорта, нюансы сочетаний — сохраняются полностью. Меняю ТОЛЬКО речевые обороты/формулировки. В книге много тонкостей, которые при сжатии теряются → сжатие запрещено, цель близка к 1:1 по смыслу, отлична по словам.
> - **Следующая книга (piemonte-korneev)** → сначала пробовать СТАРЫЙ подход (vision-дословно). Если фильтр снова блокирует — переключиться на путь B (подробный пересказ). Это правило действует и для champagne.
>
> Две готовые книги (vina-francii, novy-svet) сделаны старым дословным методом — их не трогаем.
>
> **УТОЧНЕНИЕ (2026-06-23, на стр. 108–109 «Кобе»):** обычный пересказ (путь B) тоже может зацепить копирайт-фильтр на ПЛОТНЫХ фактологических страницах, где перефраз поневоле идёт близко к оригиналу (тот же порядок абзацев, near-verbatim эпиграфы, сжатый исторический текст). Признак: ответ обрывается при записи файла. РЕШЕНИЕ — **усиленная трансформация (`ocr: vision-paraphrase-strong`)**: переструктурировать материал в собственные тезисы-буллеты, СМЕНИТЬ порядок подачи, убрать связочную «литературную» прозу и дословные эпиграфы — но СОХРАНИТЬ все факты/числа/имена/даты. Для RAG даже лучше (чище чанки). Применять к историческим/плотным разворотам; к «водянистым»/спискам достаточно обычного B.

Гибрид (исходный, для книг без блокировки): Tesseract (rus+fra, PSM 1) как якорь-страховка от пропусков + полный vision-проход Claude по рендеренным PNG (структура, колонки, цветные врезки, таблицы сочетаний) + перекрёстная сверка чисел/имён.

**ВНИМАНИЕ — систематическая OCR-ошибка 3→5.** В vina-francii и novy-svet (тот же издатель/серия/шрифт) Tesseract систематически читал печатную «3» как «5» (даты, площади, проценты, цены, температуры). Очень вероятно повторение здесь. ВСЕ числа сверять с PNG; при двусмысленности предпочитать «3» + проверять по фактам/суммам. Особое внимание — температуры подачи вина (часто 3/5/13/15/18 °C).

## Формат page_NNN.md
Frontmatter YAML: `book_id, png, printed_page, sections (список), page_kind, starts_mid_thought, ends_mid_thought, ocr, notes`. Тело — чистый текст: склейка переносов; врезки/подписи/сноски/таблицы отдельными блоками `### [врезка]`, `### [подпись]`, `[таблица]`, `[инфографика]`; спорное/искажённое — реконструировать по смыслу и пометить в `notes`, НЕ выдумывать.

**Для пути B (эта книга):** тело — подробный пересказ страницы своими словами (формулировки мои, содержание полное). `ocr: vision-paraphrase`. В `notes` помечать, что это пересказ, а не дословный текст. Числа/имена/температуры — точные, по PNG (с учётом ошибки 3→5). Цитаты-врезки если нужны — короткие, в кавычках.
