# MAIN — Распознавание книги «Вина Франции» (Simple Wine News, 2016)

> Рабочий файл-точка входа этой фичи. Читать первым при продолжении работы (в т.ч. после сжатия сессии).

## Глобальная цель

Распознать книгу в чистый структурированный текст и влить её в **единую базу знаний по вину с RAG и умным поиском**, собираемую из многих книг. Поэтому важны: точность (особенно имена/числа/аппелласьоны), провенанс (книга + страница для цитирования), и сохранность сквозных мыслей, переходящих со страницы на страницу.

## Книга

См. `meta.yaml`. Кратко: «Вина Франции. Главные сорта, регионы и аппелласьоны», серия Simple Wine News, изд-во «Э» (Эксмо), Москва 2016, ISBN 978-5-699-91519-4. PDF: 315 страниц-файлов, печатных ~312.

- **Текстовый слой PDF битый** для кириллицы (нет ToUnicode у встроенных шрифтов) → прямое извлечение бесполезно. Книга цифровая (не скан), страницы рендерятся идеально.
- **Смещение нумерации:** печатная страница = номер PNG − 1 (подтверждено по колонтитулам: file_010=стр.9, 011=10, 012=11, 013=12). НО проверять по реальному колонтитулу — могут быть ненумерованные фотополосы/заставки, сбивающие смещение.
- **Оглавлению по номерам страниц НЕ доверять** — OCR номеров в `page_006.md` ненадёжен (напр. «Массилия — 7», физически на стр. 9). `printed_page` берём из колонтитула самой страницы. TOC перегенерировать по фактическим номерам в конце.
- **Физический порядок файлов PNG = порядок чтения книги.** Книга открывается очерком «Смысл Франции в вине» + глоссарий (стр. 6–7), затем раздел «Кое-что из истории».

## Метод (гибрид, согласован с пользователем)

1. **Tesseract (rus+fra, PSM 1)** по всем страницам — *якорь верности*. Ошибается заметно (латинизация глифов, путает цифры), цветные врезки на бирюзовом/цветном фоне **теряет целиком**.
2. **Vision (Claude читает PNG)** по всем страницам — корректная структура, порядок колонок, сайдбары/подписи/сноски, французские названия. Риск — незаметное «приглаживание» текста.
3. **Сверка:** vision-текст — основной; Tesseract-якорь — для контроля чисел/имён и чтобы поймать пропуски. Расхождения в числах/именах проверять по картинке прицельно.

## Финальный результат (в git, эта папка)

- `pages/page_NNN.md` — по файлу на страницу-PNG (NNN = номер PNG, 001..315).
- `book.md` — собирается в конце: сквозные абзацы сшиты, переносы слов убраны.
- `chunks.jsonl` — нарезка для индексации (в конце).

### Формат `pages/page_NNN.md`

```
---
book_id: vina-francii
png: 12            # номер файла PNG
printed_page: 11   # номер на странице (из колонтитула); null если нет
sections: ["Религия бургундского", "Гипокрас — вино с пряностями"]
page_kind: text | photo | mixed | toc | front-matter
starts_mid_thought: false   # первый блок продолжает абзац с пред. страницы
ends_mid_thought: true      # последний блок продолжается на след. странице
ocr: vision+tesseract
---

<тело>
```

### Конвенции тела (чтобы чанкер RAG различал смысловые единицы)

- Заголовок раздела — `## Название`.
- Обычный текст — абзацами. **Переносы слов склеивать** (`вино-\nделие` → `виноделие`). Внутри страницы абзацы не рвать искусственно.
- **Сквозная мысль:** если абзац перетекает на след. страницу — не выдумывать концовку, ставить `ends_mid_thought: true`, на след. странице `starts_mid_thought: true`. Сборщик `book.md` склеит.
- **Сайдбар/врезка** (напр. бирюзовый блок «ДАТЫ») — отдельным блоком под подзаголовком `### [врезка] Название`.
- **Подпись к иллюстрации / средний блок-определение** — `> [подпись] ...`.
- **Сноска внизу полосы** — `### [сноска] Название` отдельным блоком.
- **Фото без текста** — `page_kind: photo`, тело: `*[Фотография: краткое описание]*`.
- Числа/годы/имена собственные и французские названия — переносить точно как в оригинале (это критично для RAG).
